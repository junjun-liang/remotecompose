# Remote Compose 快速上手

## 项目简介

Remote Compose 是一个 **Server-Driven UI** 演示项目，基于 AndroidX Remote Compose 技术栈，实现"服务端配置 UI，客户端原生渲染"。

核心特点：
- 通过 Web 编辑器拖拽配置界面（JSON）
- GitHub Actions 自动将 JSON 转换为二进制 `.rc` 文件
- Android App 下载 `.rc` 文件后，通过 `RemoteComposePlayer` 原生渲染
- 无需发版即可动态更新 App 界面

## 技术栈

| 模块 | 技术 |
|------|------|
| Android 播放器 | `RemoteComposePlayer` (remote-player-view) |
| 二进制格式 | Remote Compose wire format |
| JVM 转换器 | `RemoteComposeWriter` (remote-creation-core + remote-creation-jvm) |
| Web 预览 | Kotlin/Wasm + Compose Multiplatform |
| Web 编辑器 | Vanilla HTML/CSS/JS |
| CI/CD | GitHub Actions |
| 托管 | GitHub Pages |

## 本地运行

### 1. 运行 JVM 转换器（手动转换 JSON → .rc）

```bash
cd server
chmod +x gradlew

# 转换单个文件
./gradlew run --args="../config.json ../config.rc"

# 批量转换目录下所有 json
./gradlew run --args="--dir ../ ../"
```

转换完成后，同目录下会生成对应的 `.rc` 二进制文件。

### 2. 构建/运行 Web 预览（Kotlin/Wasm）

```bash
cd preview-src
chmod +x gradlew
./gradlew wasmJsBrowserRun
```

浏览器会自动打开，展示实时 Compose 预览。

### 3. 打开 Web 编辑器

直接用浏览器打开项目根目录的 `index.html`：

```bash
# macOS
open index.html
```

或把项目部署到任意静态服务器（如 GitHub Pages）。

## 工作流程

### 方式一：Web 编辑器 + GitHub Actions（推荐）

1. 打开 [Web 编辑器](https://armcha.github.io/remotecompose/) 或本地 `index.html`
2. 拖拽/编辑 UI 元素，配置颜色、文字、点击事件等
3. 输入 GitHub Token，点击 **Deploy**
4. 编辑器将 JSON 推送到仓库，触发 GitHub Actions
5. Actions 自动运行 `server` 模块，生成 `.rc` 二进制文件并提交回仓库
6. Android App 下次启动时下载最新的 `.rc` 文件并渲染

### 方式二：本地手动转换

1. 直接编辑 `config.json` 或 `config_detail.json`
2. 本地运行 `./gradlew run --args="--dir ..."` 生成 `.rc`
3. 将 `.rc` 文件放到 App 可访问的服务器/CDN
4. Android App 下载并渲染

## 关键源码说明

### server/Main.kt

CLI 入口，支持两种模式：
- `remotecompose-server <input.json> [output.rc]` — 单文件转换
- `remotecompose-server --dir <input-dir> <output-dir>` — 批量转换

### server/DocumentBuilder.kt

使用 `RemoteComposeWriter` 将 `LayoutConfig` 序列化为二进制字节数组：

```kotlin
val writer = RemoteComposeWriter(platform, HTag(DOC_WIDTH, width), HTag(DOC_HEIGHT, height))
writer.root {
    writer.column(mod, ColumnLayout.CENTER, arrangement) {
        // 递归渲染 elements
    }
}
return writer.encodeToByteArray()
```

### preview-src/PreviewRenderer.kt

Compose Multiplatform 实现的实时预览，与 Android 端渲染逻辑保持一致，方便在浏览器中即时查看效果。

## 常见问题

**Q: 为什么需要 `.rc` 二进制文件，而不是直接用 JSON？**

A: 二进制格式体积更小、解析更快，且 Android 端无需依赖 JSON 库和创建端代码，做到"纯播放器"。

**Q: Android App 如何获取 `.rc` 文件？**

A: 通常托管在 CDN/GitHub Pages，App 启动时通过 HTTP 下载。本项目示例仓库中的 `.rc` 文件可直接作为静态资源使用。

**Q: 如何添加新的界面屏幕？**

A: 1) 在 `index.html` 的 `screens` 对象中新增配置；2) 创建对应的 `config_xxx.json`；3) 在 Android 播放器中处理对应导航逻辑。
